### `issue_refund_436157` Function

```python
issue_refund_436157(order_id: str, amount: float, reason: str) -> dict
```

**Description:**

Issue a refund for a completed order.  This function simulates issuing a refund and returns a confirmation record with the refund transaction ID.

**Parameters:**

*   `order_id` (str): Identifier of the order to refund.
*   `amount` (float): Refund amount (must be > 0).
*   `reason` (str): Human-readable reason for the refund (for audit trail).

**Returns:**

`dict` with `refund_id`, `order_id`, `amount`, `reason`, and `status` keys. The status is always "refunded".

**Raises:**

*   `ValueError`: if the refund `amount` is not positive.

**Example:**

```python
refund = issue_refund_436157(order_id="12345", amount=25.00, reason="Customer return")
print(refund)
# Expected output (refund_id will vary):
# {'refund_id': 'a1b2c3d4-e5f6-7890-1234-567890abcdef', 'order_id': '12345', 'amount': 25.0, 'reason': 'Customer return', 'status': 'refunded'}
```
